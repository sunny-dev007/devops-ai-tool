## ✅ **THE FIX - Semicolon-Aware Regex**

### **Before (Broken):**

```javascript
// This regex captured EVERYTHING after "az webapp create"
const webappCreatePattern = /az\s+webapp\s+create(?:\s+\\)?\s*\n?(?:[^\n]*(?:\\|\n))*[^\n]*/gm;
```

**Problem:** No boundary detection, kept matching until end of line/file!

### **After (Fixed):**

```javascript
// This regex STOPS at semicolons and respects command boundaries
const webappCreatePattern = /az\s+webapp\s+create(?:\s+--[a-z-]+(?:\s+(?:"[^"]*"|'[^']*'|\$[A-Za-z_][A-Za-z0-9_]*|\$\{[^}]+\}|[^\s;\\]+))?)*(?:\s+\\(?:\s*\n\s*--[a-z-]+(?:\s+(?:"[^"]*"|'[^']*'|\$[A-Za-z_][A-Za-z0-9_]*|\$\{[^}]+\}|[^\s;\\]+))?)*)*/gm;
```

**Key Features:**
- ✅ Matches `--parameter-name value` patterns
- ✅ Supports quoted values: `"value"` or `'value'`
- ✅ Supports variables: `$VAR_NAME` or `${VAR_NAME}`
- ✅ Supports multi-line with backslash: `\ \n --param value`
- ✅ **STOPS at semicolon `;`** - CRITICAL!
- ✅ **STOPS at newline without backslash** - CRITICAL!
- ✅ Preserves surrounding script structure

## 🔧 **Technical Details**

### **Regex Breakdown:**

```javascript
/
  az\s+webapp\s+create                    // Match "az webapp create"
  
  (?:                                      // Start parameter group (non-capturing)
    \s+--[a-z-]+                          //   Match "--parameter-name"
    (?:                                    //   Start value group (non-capturing)
      \s+(?:                              //     Match space and one of:
        "[^"]*"                           //       - Double-quoted string
        |'[^']*'                          //       - Single-quoted string
        |\$[A-Za-z_][A-Za-z0-9_]*        //       - Variable: $VAR_NAME
        |\$\{[^}]+\}                      //       - Variable: ${VAR_NAME}
        |[^\s;\\]+                        //       - Unquoted value (STOP at ; or \)
      )
    )?                                     //   Value is optional
  )*                                       // Match 0 or more parameters
  
  (?:                                      // Start continuation group (non-capturing)
    \s+\\                                 //   Match backslash continuation
    (?:                                    //   Start continuation param group
      \s*\n\s*--[a-z-]+                   //     Match newline and "--parameter"
      (?:                                  //     Start continuation value group
        \s+(?:                            //       Match space and value
          "[^"]*"                         //         (same options as above)
          |'[^']*'
          |\$[A-Za-z_][A-Za-z0-9_]*
          |\$\{[^}]+\}
          |[^\s;\\]+                      //         CRITICAL: Stop at ;
        )
      )?
    )*                                     //   Match 0 or more continuation params
  )*                                       // Match 0 or more continuations
/gm
```

### **Key Boundary Detectors:**

1. **`[^\s;\\]+`** - Matches value BUT stops at:
   - Space (parameter boundary)
   - Semicolon `;` (command separator) **← CRITICAL**
   - Backslash `\` (continuation)

2. **No greedy `.+` or `.*`** - Each part is precisely defined

3. **Non-capturing groups `(?:...)`** - Efficient matching without capture overhead
